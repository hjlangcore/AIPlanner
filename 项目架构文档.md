## 1. 项目概述

### 1.1 项目简介

**AIPlanner** 是一款基于 AI Agent 技术的现代化待办事项与日程管理系统。它利用大语言模型（LLM）的自然语言处理能力，将用户碎片化的意图（如"下周五前完成项目报告"）自动解析为结构化的任务，并进行智能拆解、优先级评估和日程规划。

### 1.2 项目定位

- **目标用户**：需要高效管理日常任务和日程的个人用户、团队
- **核心价值**：将自然语言转换为结构化任务，减少手动录入成本
- **差异化优势**：AI 驱动的智能任务拆解、多轮对话助手、知识库增强

### 1.3 主要功能

| 功能模块 | 功能描述 |
|---------|---------|
| 智能任务解析 | 自然语言输入，自动提取任务核心、截止日期和分类 |
| 智能任务拆解 | 根据任务复杂度，自适应拆解为多个可执行的子任务 |
| 多视图展示 | 仪表盘、列表视图、日历视图，满足不同场景需求 |
| 智能提醒 | 多通道提醒（弹窗、浏览器通知、邮件） |
| 智能对话助手 | 多轮对话、意图识别、知识库增强（RAG） |
| 数据管理 | 多格式导出（JSON/CSV）、备份与恢复 |

### 1.4 技术亮点

1. **LangGraph 工作流**：基于状态图的 Agent 编排框架，支持复杂的任务处理流程
2. **流式输出 (SSE)**：实时展示 AI 分析过程，提升用户体验
3. **RAG 知识增强**：结合本地知识库，提供更精准的 AI 回答
4. **JWT 认证**：安全的多用户认证体系
5. **APScheduler 定时任务**：可靠的任务到期检测与提醒

## 4. 核心功能实现

### 4.1 智能任务解析（LangGraph Workflow）

**工作流程定义**：

```
用户输入 → process_task_node → save_to_db_node → 结束
              ↓
         ┌─────────────────────────┐
         │   调用 LLM 一次性完成    │
         │  • 提取任务名称/截止日期  │
         │  • 拆解子任务           │
         │  • 评估优先级           │
         │  • 生成日程安排         │
         └─────────────────────────┘
```

**核心代码路径**：`backend/agent/workflow.py`

**流式处理流程**：

```
1. analyzing  → AI正在分析任务...
2. parsing    → 正在解析AI响应...
3. subtask    → 逐个生成子任务
4. saving     → 正在保存任务...
5. completed  → 任务创建完成！
```

### 4.2 智能对话系统

**对话流程**：

```
用户消息 → 意图识别 → 知识检索(RAG) → 回复生成 → 保存历史
             ↓
    ┌─────────────────────┐
    │ 支持的意图：         │
    │ • create_task       │
    │ • query_task       │
    │ • general_chat     │
    │ • help             │
    └─────────────────────┘
```

**核心代码路径**：`backend/agent/chat_agent.py`

### 4.3 RAG 知识增强

**检索策略**：

1. **语义检索**：使用 TF-IDF 向量化 + 余弦相似度
2. **关键词检索**：分词 + 停用词过滤
3. **标题匹配**：精确匹配加分

**综合评分**：`semantic * 0.4 + keyword * 0.4 + title_exact * 0.2`

**核心代码路径**：`backend/service/rag_service.py`

### 4.4 定时提醒服务

**调度策略**：

| 时间点 | 任务 |
|-------|------|
| 09:00 | 每日早间巡检 |
| 14:00 | 每日午间巡检 |
| 每30分钟 | 常规检查 |

**提醒渠道**：

- 前端实时弹窗
- 浏览器通知
- 邮件推送（需配置 SMTP）

**核心代码路径**：`backend/service/reminder_service.py`

### 4.5 邮件通知

**邮件内容**：

- HTML 模板，响应式设计
- 逾期任务高亮显示
- 优先级颜色标签

**核心代码路径**：`backend/service/email_service.py`

### 4.6 数据备份与恢复

**备份格式**：ZIP 压缩包（内含 `data.json`）

**支持操作**：

- 手动/自动备份
- JSON 导出
- CSV 导出（任务）
- 从备份恢复

**核心代码路径**：`backend/service/backup_service.py`

## 7. 数据库设计

### 7.1 数据库概览

- **数据库类型**：SQLite3
- **数据库文件**：`todo_agent.db`
- **位置**：`backend/todo_agent.db`

### 7.2 表结构

#### 7.2.1 任务表 (tasks)

```sql
CREATE TABLE tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    raw_task TEXT NOT NULL,          -- 原始自然语言任务
    sub_tasks TEXT NOT NULL,         -- 拆解后的子任务（JSON数组）
    priority TEXT NOT NULL,          -- 优先级（高/中/低）
    deadline TEXT,                    -- 截止日期（YYYY-MM-DD）
    schedule TEXT,                    -- 生成的日程安排
    status TEXT DEFAULT '待执行',    -- 状态（待执行/进行中/已完成）
    category TEXT DEFAULT '默认',     -- 任务分类
    tags TEXT DEFAULT '[]',          -- 标签列表（JSON数组）
    progress INTEGER DEFAULT 0,      -- 完成进度（0-100）
    notes TEXT DEFAULT '',           -- 备注信息
    update_time TEXT,                -- 更新时间
    create_time TEXT                -- 创建时间
);
```

**索引**：

| 索引名 | 字段 | 用途 |
|-------|-----|------|
| idx_tasks_status_priority | status, priority | 状态+优先级筛选 |
| idx_tasks_category | category | 分类筛选 |
| idx_tasks_deadline_status | deadline, status | 截止日期筛选 |
| idx_tasks_create_time | create_time DESC | 排序 |

#### 7.2.2 用户表 (users)

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    full_name TEXT,
    email TEXT,
    role TEXT DEFAULT 'user',        -- 角色（admin/user）
    created_at TEXT
);
```

#### 7.2.3 分类表 (categories)

```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    color TEXT DEFAULT '#3b82f6',
    icon TEXT DEFAULT '📁',
    created_time TEXT
);
```

## 12. 常见问题与解决方案

### 12.1 后端问题

#### Q1: 启动报错 "LLM_API_KEY not found"

**原因**：未配置环境变量

**解决方案**：

1. 在项目根目录创建 `.env` 文件
2. 添加 `LLM_API_KEY=your_api_key`
3. 重启后端服务

#### Q2: 数据库锁定错误 "database is locked"

**原因**：SQLite 并发写入冲突

**解决方案**：

```python
# 在 db_handler.py 中使用检查_same_thread=False
self.conn = sqlite3.connect(db_name, check_same_thread=False)
```

#### Q3: 任务创建超时

**原因**：LLM 响应时间过长

**解决方案**：

1. 增加 API 超时配置
2. 检查网络连接
3. 考虑使用更快的模型

### 12.2 前端问题

#### Q4: API 请求 401 未授权

**原因**：Token 过期或未登录

**解决方案**：

```typescript
// 检查 localStorage 中的 token
const token = localStorage.getItem('token')
if (!token) {
  // 引导用户登录
}
```

#### Q5: 跨域请求失败

**原因**：开发环境未配置代理

**解决方案**：确保 vite.config.ts 中的代理配置正确

```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true
  }
}
```

#### Q6: 状态不同步

**原因**：缓存导致的数据过期

**解决方案**：

```typescript
// 强制刷新
await fetchTasks(true)
```

### 12.3 部署问题

#### Q7: 生产环境静态资源 404

**原因**：未正确配置 SPA 路由 fallback

**解决方案**（Nginx）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

#### Q8: 邮件发送失败

**原因**：SMTP 配置错误或防火墙阻止

**解决方案**：

1. 检查 SMTP 配置
2. 确认端口（587）开放
3. 使用测试邮件功能验证

### 12.4 性能问题

#### Q9: 首次加载慢

**原因**：依赖包过大

**解决方案**：

1. 使用 `npm run build` 生产构建
2. 启用 Gzip 压缩
